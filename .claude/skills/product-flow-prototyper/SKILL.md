# Product Flow Prototyper

Design complete, usable and accessible product flows.

## Goal

Create flows that guide the user from intention to outcome with clarity, low friction and strong feedback.

## Common flows

- onboarding
- login and signup
- password recovery
- dashboard first use
- search and filters
- checkout
- create/edit/delete
- import/export
- permissions
- settings
- upgrade/downgrade
- cancellation
- empty state recovery
- error recovery

## Required flow states

For every flow, define:

- entry point
- user goal
- success path
- error paths
- empty states
- loading states
- confirmation states
- cancellation/undo behavior
- mobile behavior
- accessibility requirements

## UX principles

- Make the next step obvious
- Reduce cognitive load
- Explain consequences before critical actions
- Give immediate feedback
- Avoid dead ends
- Use progressive disclosure
- Keep microcopy simple
- Preserve trust

## Output format

Deliver:

1. flow objective
2. user context
3. screen sequence
4. main interactions
5. edge cases
6. states
7. accessibility notes
8. implementation notes
