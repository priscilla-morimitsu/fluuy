# Mobile First Responsive Designer

Design responsive interfaces that work well across screen sizes.

## Principles

- Start with mobile constraints
- Prioritize essential content
- Avoid hover-only interactions
- Use comfortable touch targets
- Keep forms simple
- Collapse navigation thoughtfully
- Preserve context between breakpoints
- Avoid horizontal scrolling
- Keep typography readable

## Checkpoints

Review:

- mobile layout
- tablet layout
- desktop layout
- navigation behavior
- card stacking
- table adaptation
- form layout
- modal/sheet behavior
- sticky actions
- spacing changes

## Liquid Glass responsive rules

- Reduce blur on small screens if readability suffers
- Avoid stacked transparent surfaces that create noise
- Keep contrast high
- Keep glass subtle
- Do not use glow

## Output format

Deliver:

1. responsive strategy
2. breakpoint behavior
3. layout changes
4. component-specific adjustments
5. risks and fixes
