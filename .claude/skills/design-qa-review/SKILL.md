# Design QA Review

Perform a final quality review of visual design, usability, accessibility and consistency.

## Checklist

Check:

- Does the interface follow the design system?
- Do colors come from tokens?
- Are fonts correct?
- Does radius follow the standard?
- Is Liquid Glass used without glow?
- Is contrast sufficient?
- Is the screen easy to understand?
- Is the primary action clear?
- Are loading, error and empty states present?
- Does the layout work on mobile?
- Is keyboard focus visible?
- Is there excessive information?
- Is there inconsistency between components?
- Does implementation use shadcn/ui when possible?
- Are Liquid Glass effects subtle and useful?

## Approval criteria

Approve only if:

- no glow is present
- no visual style breaks the design system
- accessibility is not compromised
- the UI is clear and usable
- the implementation is maintainable

## Expected output

Deliver:

1. approved or rejected
2. main problems
3. mandatory improvements
4. optional improvements
5. final checklist
