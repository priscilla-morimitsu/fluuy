# Accessibility Auditor WCAG

Review interfaces with focus on accessibility and inclusive usability.

## Always check

- Text contrast
- Contrast inside glass components
- Keyboard navigation
- Visible focus
- Input labels
- Alternative text
- Clickable/tappable target size
- Clear error messages
- Understandable disabled states
- Screen reader compatibility
- Semantic hierarchy
- Color not being the only way to communicate status

## Rules

- Never sacrifice legibility for aesthetics
- Liquid Glass must not reduce contrast
- Every interactive element needs a focus state
- Every critical action needs clear confirmation
- Every error must explain the problem and how to fix it
- Design for users with low vision, motor limitations and cognitive load

## Expected output

Deliver:

1. problems found
2. user impact
3. recommended correction
4. priority: high, medium or low
5. acceptance criteria
