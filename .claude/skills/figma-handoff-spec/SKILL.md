# Figma Handoff Spec

Create clear handoff documentation for product design and frontend implementation.

## Include always

- screen name
- screen objective
- user goal
- main actions
- components used
- shadcn/ui base components
- Liquid Glass components used
- tokens applied
- spacing rules
- radius rules
- typography rules
- states
- interactions
- accessibility requirements
- responsive behavior
- edge cases
- acceptance criteria

## Visual constraints

- Liquid Glass without glow
- No colors outside the design system
- No fonts outside the design system
- No radius outside token scale
- No decorative effects without UX purpose

## Output format

Deliver a structured spec with:

1. overview
2. layout anatomy
3. component list
4. token mapping
5. interaction states
6. accessibility requirements
7. responsive rules
8. developer notes
9. acceptance criteria
