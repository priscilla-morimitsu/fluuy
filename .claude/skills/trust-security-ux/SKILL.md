# Trust Security UX

Design experiences that feel trustworthy and protect users from mistakes.

## Use for

- login
- signup
- authentication
- password recovery
- account deletion
- billing
- checkout
- permissions
- privacy settings
- sensitive data
- destructive actions
- session management
- exports and imports
- admin actions

## Principles

- Make consequences clear
- Do not hide important costs or risks
- Confirm destructive actions
- Provide undo when possible
- Explain why permissions are needed
- Avoid exposing sensitive details unnecessarily
- Use calm, precise language
- Never use dark patterns

## Security UX checklist

Check:

- Is the action reversible?
- Does the user understand consequences?
- Is confirmation required for dangerous actions?
- Are errors safe and not overly revealing?
- Are privacy choices understandable?
- Is billing transparent?
- Is trust built through clarity, not decoration?

## Output format

Deliver:

1. trust risks
2. security-sensitive moments
3. recommended UX improvements
4. safer microcopy
5. acceptance criteria
